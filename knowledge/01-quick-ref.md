# 搭+ 快速参考

## 产品速览

| 维度 | 内容 |
|------|------|
| 产品名称 | 搭+ (Da+) |
| Slogan | 「趣探索，轻社交，连接青年新生活」/「多样青春·搭+」 |
| 定位 | 面向青年群体的微信小程序活动报名与轻社交平台 |
| 当前版本 | V1.4.11 |
| 产品形态 | 微信小程序(C端+B端) + 后台管理系统 + PC端管理后台 |
| 支付方案 | 拉卡拉分账模式（用户支付→冻结→分账至主办方~90%+平台~10%） |
| 审核机制 | 零审核「即发即上」模式（从V1.0人工审核演进而来） |

## 用户角色

| 角色 | 能力 | 限制 |
|------|------|------|
| 个人用户 | 发布普通活动、报名、支付、核销 | 不能发布比赛 |
| 认证主理人 | 发布比赛、赛事管理、售后操作 | 售后仅限PC端后台 |
| 平台管理员 | 后台管理、活动管理、会员管理 | - |

身份标识：手机号为唯一身份标识。主理人认证流程：未认证→审核中→已认证。

## 系统架构

```
前端(小程序): 首页/活动列表、活动详情、发布、我参与的、钱包、消息、个人中心、俱乐部、路线发现、潮玩榜单
    ↕
API层: RESTful接口
    ↕
服务层: 用户/主办方/活动/订单/支付/核销/售后/财务/消息/分类标签管理
    ↕
数据层: MySQL + 云存储 + AES加密
    ↕
后台: 主理人管理、活动管理、订单管理、财务管理、会员管理、系统配置
PC端: 赛事管理（成绩录入、赛程编排、售后操作）
```

## 核心业务流程

**主办方**: 发布活动(零审核) → 用户报名支付 → 查看名单 → 现场扫码核销 → 活动结束 → 结算(线下)

**用户**: 浏览活动 → 选择报名 → 填写信息 → 微信支付 → 生成核销二维码 → 凭码核销入场 → 评价

## 6 个关键架构决策

1. **零审核** — 从人工审核演进为即发即上，降低B端摩擦
2. **比赛独立业务线** — 不与普通活动混用，共享基础架构但独立功能
3. **拉卡拉分账结算** — 用户支付→中间账户冻结→分账至主办方+平台→各自提现，2.5‰手续费平台承担
4. **区域筛选按行政区划** — 按县区而非车程/距离（因手中有县区资源）
5. **T-30滚动窗口排行榜** — 每日凌晨重算，仅统计过去30天，避免固化
6. **目的地指南轻量化** — 富文本+灵活分类，不建复杂POI数据库
