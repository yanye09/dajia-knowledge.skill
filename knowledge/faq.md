# 搭+ 常见问题速查

> Skill 触发后，先扫描此文件。命中问题则直接回答，无需加载其他 knowledge 文件。
> 每个回答标注 **[来源]**，便于追溯。

## 产品基础

### Q1: 搭+是什么？
**[已确认]** 面向青年群体的微信小程序活动报名与轻社交平台，连接主办方（主理人）与参与者。Slogan：「趣探索，轻社交，连接青年新生活」。[来源: 01-quick-ref.md]

### Q2: 当前版本号？
**[已确认]** V1.4.11。[来源: 00-meta.md]

### Q3: 搭+用什么支付？
**[已确认]** 拉卡拉分账模式。用户微信支付→资金冻结在拉卡拉中间账户→活动结束后平台发起分账→主办方约90%、平台约10%分别入账→各自提现。交易手续费2.5‰由平台承担。拉卡拉平台分账接收账号: SR2024000131998。[来源: 03-rules.md + 拉卡拉分账详解]

### Q4: 审核机制是怎样的？
**[已确认]** 零审核「即发即上」模式。V1.0 原为人工审核流程，后演进为零审核以降低 B 端发布摩擦。后台可配置自动审核开关。[来源: 01-quick-ref.md]

## 功能

### Q5: 有没有比赛功能？
**[已确认]** 有。比赛作为独立业务线已上线（V1.3），不与普通活动混用。包含 4 条功能线：赛事发布与组别管理、C 端报名、B 端赛事管理、赛后管理。团队报名不做，电子完赛证书不要了。[来源: 02-features.md]

### Q6: 支持退款吗？
**[已确认]** 支持。活动结束后 7 日内可发起一次性售后申请，支持部分名额退款。主办方可确认退款/修改金额/拒绝。超时 7 天未处理→挂起→平台介入。微信退款原路返回，1-3 工作日到账。[来源: 03-rules.md]

### Q7: 有没有俱乐部功能？
**[已确认]** 有。俱乐部生态已在 V1.2 上线：入驻认证、俱乐部主页、列表与分类浏览、入驻欢迎海报、分享转发。点击发布者头像/名称可跳转俱乐部详情。[来源: 02-features.md]

### Q8: 有没有评价系统？
**[已确认]** 待开发。已设计完成：三档满意度（1/3/5 分）+ 快速标签 + 文字评论 + 图片上传，含防骚扰机制和积分激励。但尚未上线。[来源: 02-features.md]

### Q9: 有没有装备商城？
**[已确认]** 待开发，规划中。[来源: 02-features.md]

### Q10: 有没有数据看板？
**[已确认]** 待开发（真实数据看板在规划中）。B 端赛事管理已有基础数据看板（报名人数/金额/组别统计）。财务流水月报表为 P0 开发中。[来源: 02-features.md + 04-versions.md]

## 角色与权限

### Q11: 怎么成为主理人？
**[已确认]** 个人身份点击「发布比赛」→ 弹窗引导认证 → 提交认证 → 审核中/已认证。认证通过后可发布比赛、使用赛事管理工具。售后操作仅限 PC 端后台。[来源: 03-rules.md]

### Q12: 个人用户可以发布比赛吗？
**[已确认]** 不可以。点击「发布比赛」会触发认证引导弹窗。只有认证主理人才可以发布比赛。[来源: 03-rules.md]

## 业务规则

### Q13: 排行榜怎么算的？
**[已确认]** T-30 滚动窗口，每日凌晨 00:00 重算。俱乐部榜 = 30 天成功支付订单数 / 发布活动场次数。用户榜 = 30 天累计成功支付订单数。前端展示分 = 真实热度 + 虚拟热度加成（后台手动填写，默认 0）。[来源: 03-rules.md]

### Q14: 区域筛选怎么用？
**[已确认]** B 端发布时省市区三级级联选择器强必填。C 端首页顶部区域 Tab，全部固定首位其余按活动数降序，无在架活动的县区自动隐藏。某区域 < 3 个活动→缺省文案 + 跨区热门推荐填充。[来源: 03-rules.md]

### Q15: 财务结算是线上的吗？
**[已确认]** 是。采用拉卡拉分账模式，全流程线上化：用户支付→拉卡拉中间账户冻结→活动结束平台发起分账指令→资金自动分配至主办方和平台各自拉卡拉账户→提现。平台收入在分账完成时确认（权责发生制），按技术服务费缴纳6%增值税。退款采用「先分账后冲正」流程。[来源: 03-rules.md + 拉卡拉分账详解]

## 技术

### Q16: 技术栈是什么？
**[已确认]** 前端：微信小程序原生。后端：RESTful API + MySQL。支付：拉卡拉。认证：手机号+密码登录（V1.3.15 新增）。安全：AES 加密敏感字段，身份证后台脱敏展示。[来源: 05-tech.md]
