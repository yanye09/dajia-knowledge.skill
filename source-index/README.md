## 索引规则

1. **首次索引**: 用户提供代码库位置后，扫描目录树，自动填充上表
2. **增量更新**: 代码库有新增/删除文件时，手动更新对应映射
3. **触发时机**: 用户说「更新源码索引」「扫描代码库」或代码库发生结构性变化时
4. **注意**: 源码索引只记录文件位置和模块归属，不存储源码内容

## 使用方法

当用户询问「XX 功能的代码在哪里」时：
1. 查找本文件中的功能→文件映射
2. 提供具体的文件路径
3. 如果映射不存在，回答：「源码索引中尚未记录该功能的代码位置，需要先扫描代码库更新索引」
